# BSV 架构与构建说明

## 模块职责

- **core_types**: 定义像素格式、缓冲描述、状态机与错误码等基础类型。
- **buffer**: 提供跨平台缓冲抽象与平台句柄封装，支持外部句柄导入与内部创建。
- **camera provider**: 负责采集与帧回调，目前实现 Android Camera2 与 Linux 侧抽象。
- **gpu csc**: 负责 NV12/NV21 到 RGBA8888 的 GPU 颜色空间转换，Linux 与 Android 共享 shader。
- **controller**: 管理生命周期与线程安全，协调 camera provider 与 CSC 处理管线。
- **tests**: 提供 Linux 场景与单元测试，验证构建、EGL 初始化与转换正确性。

## 构建方式

### 使用 CMakePresets (推荐)

- Linux:
  ```bash
  cmake --preset linux
  cmake --build --preset linux
  ```
- Android (NDK r25c):
  ```bash
  cmake --preset android
  cmake --build --preset android
  ```

### 传统 CMake 命令

- Linux:
  ```bash
  cmake -S . -B build/linux -DCMAKE_BUILD_TYPE=Debug -DBSV_BUILD_TESTS=ON
  cmake --build build/linux
  ```
- Android:
  ```bash
  cmake -S . -B build/android \
    -DCMAKE_BUILD_TYPE=Debug \
    -DBSV_BUILD_TESTS=OFF \
    -DCMAKE_TOOLCHAIN_FILE=/opt/android-ndk-r25c/build/cmake/android.toolchain.cmake \
    -DANDROID_NDK=/opt/android-ndk-r25c \
    -DANDROID_ABI=arm64-v8a \
    -DANDROID_PLATFORM=android-24
  cmake --build build/android
  ```

## 平台差异

- **缓冲类型**:
  - Linux 侧使用主机内存与 EGL/GLES 进行 GPU CSC。
  - Android 侧使用 AHardwareBuffer 作为平台句柄，并通过 EGLImage 进行 GPU 渲染。
- **图形上下文**:
  - Linux 使用 EGL PBuffer 或 surfaceless display 初始化 GLES。
  - Android 通过 NDK EGL 与 GLES2/3 初始化，优先尝试 GLES3。
- **摄像头实现**:
  - Android 使用 Camera2 + AImageReader。
  - Linux 目前为抽象接口，具体实现由上层集成。
